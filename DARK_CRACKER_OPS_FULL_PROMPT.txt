penetration testing platform built entirely in Python with a PyQt5 GUI.

attack pipeline, AI-assisted analysis, scheduled attacks, packet capture,

  SECTION 6 — THE 9-STAGE ATTACK PIPELINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STAGE 1 — SCAN
  Tool: airodump-ng
  Action: Puts interface into monitor mode, scans all channels
  Output: List of networks (BSSID/SSID/channel/encryption/signal/clients)
  Duration: Configurable (default 60 seconds)
  Result → populates NetworkTable, SignalGraph

STAGE 2 — CAPTURE
  Strategy selected based on target encryption:

  WPA/WPA2 — PMKID Attack (preferred, clientless):
    Tool: hcxdumptool → hcxpcapngtool
    Action: Captures PMKID from AP beacon frames (no client needed)
    Output: .hc22000 hash file

  WPA/WPA2 — Handshake Capture (if PMKID fails):
    Tool: airodump-ng + aireplay-ng
    Action: Monitors for EAPOL 4-way handshake, sends deauth to force reconnect
    Output: .cap file with handshake

  WEP Attack:
    Tool: airodump-ng + aireplay-ng (ARP replay, fragmentation)
    Action: Collects IVs, cracks when sufficient count reached
    Output: WEP key directly

  WPA3-SAE:
    Tool: hcxdumptool
    Action: Captures SAE commit frames for analysis
    Note: MT7601U adapter may have limited injection support

  WPS Attack (if WPS enabled):
    Tool: reaver (pixie-dust) or wash
    Action: Tries WPS PIN brute force / pixie-dust attack
    Output: WPS PIN + WPA password

STAGE 3 — CRACK
  Primary: hashcat mode 22000 (WPA-PMKID/HANDSHAKE)
    hashcat -m 22000 hash.hc22000 wordlist.txt
  Fallback: aircrack-ng -w wordlist.txt -b BSSID capture.cap
  Progress: real-time speed (H/s) + ETA display in terminal
  Wordlist mutations: 50+ SSID-based variations auto-generated

STAGE 4 — CONNECT
  Tool: nmcli
  Action: Connects to target network using cracked password
  Verifies: actual IP assignment, internet connectivity check

STAGE 5 — DISCOVER
  Tool: arp-scan (primary) + nmap ping sweep (fallback)
  Action: Discovers all live hosts on the connected subnet
  Output: IP / MAC / hostname / vendor for each host
  Result → populates HostsTable, NetworkMapWidget

STAGE 6 — PORTS
  Tool: nmap -sV -sC -O --open -T4
  Action: Service version detection + OS fingerprinting on all discovered hosts
  Risk scoring: CRITICAL/HIGH/MEDIUM/LOW/INFO per port
  Result → populates PortTable

STAGE 7 — ENUM
  Service-specific enumeration:
  Web (80/443/8080/8443): nikto + gobuster directory scan
                          + sensitive path probe (/.env, /admin, etc.)
  SSH (22):               Banner grab, key exchange info
  FTP (21):               Anonymous login test, file listing
  SMB (445/139):          Null session, share enumeration (smbclient + nmap NSE)
  Databases (3306/5432/27017): Banner grab, version check

STAGE 8 — EXPLOIT
  CVE lookup: queries NVD API for each discovered service/version
              runs searchsploit if installed
  Default creds: tries 100+ default username/password pairs across
                 SSH / FTP / Telnet / HTTP / MySQL / MSSQL / RDP / SNMP
                 (7 service types × 10-15 cred pairs each)
  SSH brute: paramiko + sshpass fallback
  FTP brute: ftplib, anonymous test first, then credential list
  SMB: smbclient null session, nmap NSE vuln scripts
  Result → populates CredsPanel, updates VulnerabilitiesTable

STAGE 9 — REPORT
  Generates professional report with all findings
  Formats: HTML / PDF (reportlab) / JSON / TXT
  Audiences: Technical (full detail) or Executive (summary)
  Report sections:
    Cover page, Executive Summary, Risk Scorecard (A-F grade per host),
    WiFi Networks Found, Bluetooth Devices, Network Hosts,
    Vulnerability Findings (sorted by CVSS score),
    Open Ports & Services, Discovered Credentials (passwords starred),
    Attack Timeline, Remediation Plan, Recommendations, Technical Appendix
  Attack narrative auto-generated from timeline events
  Saved to: ~/.darkcracker/reports/

1. Lazy Tab Loading
   Tabs 9-13 (Packet Capture, C2, OSINT, Scheduler, Marketplace)
   show a placeholder until first clicked, then load the real widget.
   Reduces startup time from ~2s to ~0.3s.

2. Timer Pause on Hide
   SignalGraph and NetworkMapWidget stop their QTimers when the tab
   is not visible. Reduces background CPU by ~40%.

3. Deferred AI Knowledge Base
   ai_model.py imports in 0.011s. The 200+ entry KB is built only
   on the first query call, not at import time.

4. QTableView + QStandardItemModel
   All three main tables (NetworkTable, HostsTable, PortTable) use
   QTableView with model-view separation. Handles 10,000+ rows smoothly.
   QSortFilterProxyModel provides free column-click sorting.

5. Database Indexes
   6 indexes on foreign-key columns (session_id, host_id, target).
   Makes history/report queries instant on large datasets.

6. Terminal Line Cap
   TerminalWidget auto-trims to 5000 lines maximum.
   Prevents memory leak during long attack sessions.

7. Batch Table Updates
   begin_batch() / end_batch() methods on all tables.
   Blocks signals and disables repaints during bulk inserts.

8. Smooth Splash Animation
   60fps (16ms QTimer) progress bar interpolation on splash screen.
   80ms step interval (down from 120ms).

STEP 1 — Launch
  sudo python3 main.py
  Splash screen animates through 8 loading steps (0.64s total)
  MainWindow opens, database initialized, interface auto-detected

STEP 2 — Interface Setup
  Left sidebar shows wlan0 in the dropdown
  If in managed mode: click "ENABLE MONITOR" → runs airmon-ng start wlan0
  If already in monitor mode: shows cyan dot "● monitor"

STEP 3 — WiFi Scan
  Click "WIFI SCAN" (Dashboard quick launch) OR
  Set interface in left panel → click LAUNCH
  WIFI ATTACK tab activates, pipeline SCAN stage turns cyan
  airodump-ng starts, NetworkTable populates with discovered networks
  SignalGraph plots real-time signal strength

STEP 4 — Target Selection
  Click a network row → BSSID/SSID auto-fills in left panel
  Check SECURITY column: WPA2 = PMKID attack, WPS = try reaver
  Right-click → "Set as Target" or "Start Attack"

STEP 5 — Attack Launch
  Select mode in left panel (Auto recommended)
  Set wordlist path (default: /usr/share/wordlists/rockyou.txt)
  Click LAUNCH
  Pipeline advances: SCAN → CAPTURE (cyan, pulsing)
  Terminal shows live attack output

STEP 6 — Capture
  PMKID attack: hcxdumptool runs, captures PMKID from AP
  If no PMKID: falls back to handshake capture (deauth sent)
  Progress bar shows 0% → 50%
  On success: hash file ready, pipeline → CRACK (cyan)

STEP 7 — Crack
  hashcat starts with mode 22000
  Terminal shows: speed (H/s), progress %, ETA
  On crack: password displayed in terminal + toast notification
  Pipeline → CONNECT (cyan)

STEP 8 — Connect
  nmcli connects to target network with cracked password
  Verifies IP assignment
  Pipeline → DISCOVER (cyan)

STEP 9 — Host Discovery
  arp-scan sweeps the subnet
  HostsTable populates with all live hosts
  NetworkMapWidget shows topology
  Pipeline → PORTS (cyan)

STEP 10 — Port Scan
  nmap runs against all discovered hosts
  PortTable populates with services, versions, risk levels
  Pipeline → ENUM (cyan)

STEP 11 — Enumeration
  Web services: nikto + gobuster run
  FTP/SSH/SMB: default credential testing
  Results appear in terminal + CredsPanel
  Pipeline → EXPLOIT (cyan)

STEP 12 — Exploitation
  CVE lookup against discovered service versions
  SSH/FTP/SMB brute force with expanded credential list
  Any valid credentials → CredsPanel STATUS = VALID (green)
  Pipeline → REPORT (cyan)

STEP 13 — Report Generation
  Reports tab → fill engagement metadata
  Select sections to include
  Choose format (HTML/PDF/JSON/TXT)
  Click GENERATE REPORT
  Pipeline → REPORT = DONE (green)
  File saved to ~/.darkcracker/reports/
